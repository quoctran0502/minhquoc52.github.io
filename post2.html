Ngôn Ngữ C – Nền Tảng Lập Trình

Năm 1 • 2022
1

Vì sao bắt đầu bằng C?

C giúp mình “thấy” những gì diễn ra bên dưới: bộ nhớ, con trỏ, stack/heap và cách chương trình nói chuyện với hệ điều hành. Hiểu C tốt → chuyển qua C++/Java/Go dễ hơn vì đã nắm lõi.

Quản lý bộ nhớChủ động malloc/free, hiểu rò rỉ & vùng nhớ
Hiệu năngÍt trừu tượng, tối ưu gần phần cứng
Khả chuyểnBiên dịch đa nền tảng, chuẩn C
2

Biến, hàm, & con trỏ – chạm vào bộ nhớ


#include <stdio.h>

int sum(int a, int b){ return a + b; }

int main(){
  int x = 5, y = 7;
  int *p = &x;          /* p giữ địa chỉ của x */
  printf("x=%d, *p=%d, sum=%d\n", x, *p, sum(x,y));
  printf("addr x=%p, addr p=%p\n", (void*)&x, (void*)&p);
  return 0;
}

Tóm lại

  • Con trỏ là biến giữ địa chỉ → dereference bằng *p.
  • Tham trị vs tham chiếu: muốn hàm “đổi” dữ liệu gốc → truyền địa chỉ.
3

Mảng & chuỗi C – vùng nhớ liên tiếp


#include <stdio.h>
#include <string.h>

void print_arr(int *a, int n){
  for (int i=0;i<n;i++) printf("%d ", a[i]);
  printf("\n");
}

int main(){
  int a[4] = {1,2,3,4};
  print_arr(a, 4);                 // mảng decay thành con trỏ

  char s[16];                       // chuỗi C: mảng char + '\0'
  strcpy(s, "Hello");
  printf("%s (%zu)\n", s, strlen(s));

  // con trỏ số học
  int *p = a;  // trỏ tới a[0]
  printf("*(p+2)=%d\n", *(p+2)); // => a[2]
  return 0;
}

Lưu ý nhanh

  • Mảng và con trỏ liên quan nhưng không giống nhau (kích thước khác, không gán lại tên mảng).
  • Chuỗi C phải có ký tự kết thúc '\0'; cẩn thận tràn bộ đệm khi strcpy.
4

struct, typedef & quản lý bộ nhớ


#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
  char name[32];
  int  age;
} Person;

Person *make_person(const char* n, int age){
  Person *p = (Person*)malloc(sizeof(Person));
  strncpy(p->name, n, sizeof(p->name)-1);
  p->name[sizeof(p->name)-1] = '\0';
  p->age = age;
  return p;
}

int main(){
  Person *alice = make_person("Alice", 20);
  printf("%s - %d\n", alice->name, alice->age);
  free(alice); // luôn nhớ giải phóng
  return 0;
}

Gợi ý an toàn

  • Dùng strncpy/snprintf để tránh tràn bộ đệm.
  • Quy ước “ai malloc thì người đó free”.
5

Tách file .h/.c & biên dịch


// mathx.h
int add(int a, int b);

// mathx.c
#include "mathx.h"
int add(int a, int b){ return a+b; }

// main.c
#include <stdio.h>
#include "mathx.h"
int main(){ printf("%d\n", add(3,4)); }
    

# Biên dịch & liên kết
gcc -c mathx.c        # tạo mathx.o
gcc -c main.c         # tạo main.o
gcc main.o mathx.o -o app
./app                 # => 7

Kết luận

  • Hiểu con trỏ, mảng, chuỗi, struct là “xương sống” của C.
  • Tách file rõ ràng giúp dự án lớn dễ quản lý & build.
