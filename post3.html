Làm Quen Với Lập Trình Hướng Đối Tượng Trong Java

Năm 2 • 2023
1

Tư duy đối tượng

OOP gom dữ liệu và hành vi vào một đơn vị là class. Ta thao tác qua đối tượng (object) thay vì lộ chi tiết triển khai. Nhờ vậy mã dễ mở rộng, test và tái sử dụng.

Mục tiêu của OOP

  • Encapsulation: che giấu chi tiết – chỉ lộ public API.
  • Inheritance: tái sử dụng & mở rộng hành vi.
  • Polymorphism: cùng 1 lời gọi nhưng chạy đúng phiên bản (override) ở runtime.
2

Class & Đóng gói (Encapsulation)


class BankAccount {
  private double balance;       // ẩn bên trong

  public BankAccount(double init) {
    this.balance = init;
  }

  public void deposit(double amount) {
    if (amount <= 0) throw new IllegalArgumentException("amount");
    balance += amount;
  }

  public boolean withdraw(double amount) {
    if (amount <= 0 || amount > balance) return false;
    balance -= amount;
    return true;
  }

  public double getBalance() { return balance; }
}

Ghi nhớ

  • Trường dữ liệu để private, expose qua phương thức.
  • Đặt invariant (điều kiện bất biến) ngay trong class để bảo vệ trạng thái.
3

Kế thừa & Đa hình (Override)


class Vehicle {
  void move() { System.out.println("moving..."); }
}

class Car extends Vehicle {
  @Override void move() { System.out.println("car..."); }
}

class Bike extends Vehicle {
  @Override void move() { System.out.println("bike..."); }
}

public class Demo {
  public static void main(String[] a){
    Vehicle v1 = new Car();
    Vehicle v2 = new Bike();
    v1.move();  // car...
    v2.move();  // bike...
  }
}

Note

  • Dynamic dispatch: biến kiểu cha nhưng thực thi theo kiểu thật (Car/Bike).
  • Ưu tiên lập trình theo abstraction (kiểu cha/giao diện) thay vì kiểu cụ thể.
4

Interface & Abstract class


interface Payable { double pay(); }

abstract class Employee implements Payable {
  protected final String name;
  Employee(String n){ this.name = n; }
  abstract double pay();     // bắt buộc lớp con cài đặt
}

class SalariedEmp extends Employee {
  private double monthly;
  SalariedEmp(String n, double m){ super(n); monthly = m; }
  @Override double pay(){ return monthly; }
}

Khi nào dùng?

  • interface: mô tả hành vi (contract), đa kế thừa được.
  • abstract class: chia sẻ 1 phần triển khai + trạng thái chung.
5

Composition > Inheritance?


interface Engine { void start(); }
class V8Engine implements Engine { public void start(){ System.out.println("V8!"); } }

class Car2 {                // dùng composition
  private final Engine engine;
  Car2(Engine e){ this.engine = e; }
  void start(){ engine.start(); }
}

Vì sao chuộng composition?

  • Giảm phụ thuộc chặt (tight coupling) do cây thừa kế phức tạp.
  • Dễ thay đổi hành vi bằng cách truyền đối tượng khác (DI/Strategy).
6

Best-practice nhanh

  • Ưu tiên bất biến (fields final khi có thể), giới hạn mutability.
  • Override toString, equals, hashCode cho entity/vo.
  • Đặt cấu trúc package theo layer: domain, service, web.
  • Khi vào Spring: bơm phụ thuộc qua constructor, code theo interface.
7

Kết luận

OOP là nền tảng để bước vào những framework lớn như Spring. Làm chủ đóng gói, kế thừa, đa hình và biết khi nào chọn composition sẽ giúp kiến trúc sạch và dễ mở rộng.
